Keeping API Keys Safe with Cloudflare Workers

The R&R Records AI chatbot connects to three third-party APIs: DeepSeek (LLM), Last.fm (music data), and Discogs (release info). Here is how Cloudflare keeps those keys secure at every step.

1. Description — How Cloudflare Protects the Keys

The browser-based chatbot needs to call DeepSeek, Last.fm, and Discogs APIs, but embedding API keys in client-side JavaScript would expose them to anyone inspecting the page source or Network tab. Instead, a Cloudflare Worker acts as a secure proxy layer between the browser and the third-party APIs.

How the proxy pattern works

  1. The frontend (static HTML/JS) makes a request to the Worker at a path like /api/llm — with no API key anywhere in the request.
  2. The Cloudflare Worker receives the request on Cloudflare's edge network.
  3. The Worker reads the API key from Cloudflare Secrets (encrypted environment variables) via env.LLM_KEY.
  4. The Worker injects the key server-side into the Authorization header or query string and forwards the request to the real API (DeepSeek, Last.fm, or Discogs).
  5. The API response flows back through the Worker to the browser. The browser never sees the keys — not in source code, not in the Network tab, not in git history.

Three layers of protection

  • Encryption at rest: wrangler secret put encrypts the value before storing it in Cloudflare's infrastructure.
  • Server-side only: Secrets are injected into the Worker's env object at runtime — never transmitted to the client.
  • Git safety: The config.js file (used for local development) is in .gitignore. Only config.example.js with placeholder values is committed.

2. Instructions — How to Set This Up Yourself

Prerequisites

  • A Cloudflare account (free tier works)
  • Node.js and npm installed
  • API keys for the services you want to proxy

Step A — Create the Worker file

Create worker.js with routes that proxy to each API:

export default {
  async fetch(request, env, ctx) {
    const url = new URL(request.url);

    // CORS headers for your frontend
    const headers = {
      "Access-Control-Allow-Origin": "*",
      "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
      "Access-Control-Allow-Headers": "Content-Type, Authorization",
    };

    if (request.method === "OPTIONS") {
      return new Response(null, { status: 204, headers });
    }

    try {
      let response;

      if (url.pathname === "/api/llm") {
        const body = await request.json();
        response = await fetch("https://api.deepseek.com/v1/chat/completions", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            "Authorization": `Bearer ${env.LLM_KEY}`,
          },
          body: JSON.stringify(body),
        });
      } else if (url.pathname === "/api/lastfm") {
        const params = url.searchParams.toString();
        const lfmUrl = `https://ws.audioscrobbler.com/2.0/?${params}&api_key=${env.LFM_KEY}&format=json`;
        response = await fetch(lfmUrl);
      } else if (url.pathname === "/api/discogs") {
        const q = url.searchParams.get("q") || "";
        const type = url.searchParams.get("type") || "release";
        const dgsUrl = `https://api.discogs.com/database/search?q=${encodeURIComponent(q)}&type=${type}&token=${env.DGS_KEY}`;
        response = await fetch(dgsUrl, {
          headers: { "User-Agent": "MyApp/1.0" }
        });
      } else {
        return new Response("Not found", { status: 404, headers });
      }

      // Forward response with CORS
      const resHeaders = new Headers(response.headers);
      Object.entries(headers).forEach(([k, v]) => resHeaders.set(k, v));
      return new Response(response.body, {
        status: response.status, headers: resHeaders,
      });
    } catch (e) {
      return new Response(JSON.stringify({ error: e.message }), {
        status: 500,
        headers: { ...headers, "Content-Type": "application/json" },
      });
    }
  },
};

Step B — Create wrangler.toml

name = "my-api-proxy"
main = "worker.js"
compatibility_date = "2024-01-01"

Step C — Log in and store secrets

npx wrangler login          # Opens browser, log into Cloudflare
npx wrangler secret put LLM_KEY     # Paste your DeepSeek/OpenAI key
npx wrangler secret put LFM_KEY     # Paste your Last.fm key
npx wrangler secret put DGS_KEY     # Paste your Discogs token
Note: Each wrangler secret put command prompts you to paste the value. The value is encrypted and never displayed back. You can verify secrets exist with npx wrangler secret list (shows names only, not values).

Step D — Deploy

npx wrangler deploy

This will print a URL like https://my-api-proxy.yourusername.workers.dev. Save this — it is your proxy endpoint.

Step E — Wire your frontend to the proxy

In your browser-side JavaScript, replace direct API calls with calls to your Worker:

// Instead of calling DeepSeek directly with your key:
// ❌ fetch("https://api.deepseek.com/...", {
//      headers: { "Authorization": "Bearer sk-..." }
//    })

// ✅ Call your Worker — no key needed in the browser:
fetch("https://my-api-proxy.yourusername.workers.dev/api/llm", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ model: "deepseek-chat", messages: [...] })
})

Step F — Gitignore your local config

Create a config.example.js with placeholder values that you do commit, and add the real config.js to .gitignore.
